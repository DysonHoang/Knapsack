Phần 1

Knapsack DP là gì?

Quy hoạch động (Dynamic Programming) kết hợp tính đúng đắn của duyệt toàn bộ và tính hiệu quả của thuật toán tham lam. Ý tưởng: chia bài toán lớn thành các bài toán con nhỏ hơn, giải từng bài toán con và lưu kết quả lại.

Knapsack (bài toán cái ba lô) là một trong những dạng DP kinh điển và phổ biến nhất. Hầu hết mọi bài Knapsack đều có chung một ý tưởng: mỗi vật phẩm có trọng lượng dương, tổng trọng lượng không vượt quá giới hạn cho trước.

Ba biến thể thường gặp

0/1 Knapsack
Chọn tập con tối ưu
Mỗi vật phẩm chỉ dùng được 1 lần. Tìm tập con có tổng giá trị lớn nhất mà không vượt quá sức chứa.
Unbounded
Dùng không giới hạn lần
Mỗi loại vật phẩm có thể chọn nhiều lần. Đếm số cách hoặc tìm cách tối ưu để lấp đầy ba lô.
Biến thể phức tạp
Kết hợp nhiều ràng buộc
Thêm trạng thái phụ vào DP, như chia nhóm, tỉ lệ tài năng/trọng lượng, chu trình đường đua...
Phần 2

Lý thuyết cốt lõi

Bài toán gốc: Cho danh sách trọng lượng [1, 3, 3, 5], xác định tất cả các tổng có thể tạo được.

Định nghĩa hàm trạng thái

Ta định nghĩa possible(x, k) = true nếu có thể tạo ra tổng x bằng cách sử dụng k trọng lượng đầu tiên.

possible(x, k) = possible(x − wk, k−1) || possible(x, k−1)

Công thức này thể hiện: để tạo tổng x dùng k vật phẩm đầu, ta hoặc là chọn wk (thì còn lại phải tạo x − wk từ k−1 vật) hoặc không chọn wk (thì tạo x từ k−1 vật).

Cài đặt 2D (tường minh)

// possible[x][k] = true nếu tổng x đạt được bằng k vật đầu
possible[0][0] = true;
for (int k = 1; k <= n; k++) {
    for (int x = 0; x <= W; x++) {
        if (x - w[k] >= 0)
            possible[x][k] |= possible[x - w[k]][k - 1];
        possible[x][k] |= possible[x][k - 1];
    }
}

Cài đặt 1D (tối ưu bộ nhớ)

// Chỉ cần 1 mảng, duyệt ngược để tránh dùng lại vật phẩm
possible[0] = true;
for (int k = 1; k <= n; k++) {
    for (int x = W; x >= 0; x--) {
        if (possible[x]) possible[x + w[k]] = true;
    }
}

💡 Tại sao duyệt ngược? Khi duyệt x từ lớn đến nhỏ, đảm bảo mỗi vật phẩm chỉ được xét 1 lần (0/1 Knapsack). Nếu duyệt xuôi, vật phẩm k có thể được dùng nhiều lần — đó là Unbounded Knapsack.

Phần 3

Bài toán Coin Change

Đây là ứng dụng kinh điển của Knapsack: Cho n loại đồng xu, tìm số xu ít nhất để tạo đúng tổng x.

Định nghĩa trạng thái DP

dp[i] = số đồng xu ít nhất để tạo tổng i

dp[i] = min over all coins c: dp[i − c] + 1

Khởi tạo và kết quả

• dp[0] = 0 — Không cần xu nào để tạo tổng 0

• dp[i] = ∞ ban đầu — Chưa biết cách tạo tổng i

• Kết quả: dp[x] (nếu vẫn là ∞ → in −1)

Code tham khảo (C++)

#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n, x;
    cin >> n >> x;
    vector<int> coins(n);
    for (int i = 0; i < n; i++) cin >> coins[i];

    const int INF = 1e9;
    vector<int> dp(x + 1, INF);
    dp[0] = 0;

    for (int i = 1; i <= x; i++) {          // duyệt tổng
        for (int c : coins) {                 // thử từng xu
            if (i - c >= 0)
                dp[i] = min(dp[i], dp[i - c] + 1);
        }
    }
    cout << (dp[x] == INF ? -1 : dp[x]) << "\n";
}

Độ phức tạp: O(n × x) — vòng lặp ngoài chạy x lần, vòng trong duyệt n loại xu.

Phần 4

Tổng quan các bài tập

Tài liệu trình bày 8 bài có hướng dẫn và 4 bài tự luyện, phủ đầy đủ các biến thể của Knapsack.

BT 1
Minimizing Coins – Giảm thiểu số tiền xu
Unbounded Knapsack. Tìm số xu ít nhất để tạo đúng tổng x. dp[i] = min số xu để tạo i.
O(n·x)
BT 2
Coin Combinations I – Tổ hợp có thứ tự
Đếm số dãy xu (tính thứ tự) tạo ra tổng x. Duyệt tổng trước, xu sau → dp[i] += dp[i−c].
O(n·x)
BT 3
Coin Combinations II – Tổ hợp không thứ tự
Đếm số cách chọn xu (không tính thứ tự) để tạo tổng x. Duyệt xu trước, tổng sau.
O(n·x)
BT 4
Bookshop – Hiệu sách
0/1 Knapsack cổ điển. Mỗi sách mua tối đa 1 lần. Tối đa số trang trong ngân sách x. Duyệt ngược j.
O(n·x)
BT 5
Coding Company – Chia nhóm lập trình viên
DP 3 chiều: (người đã xét, số nhóm đang mở, tổng penalty). Biến thể phức tạp, sắp xếp trước.
O(n²·x)
BT 6
Fruit Feast – Bữa tiệc trái cây
Unbounded + 1 biến cố (uống nước 1 lần). DP 2 trạng thái: chưa uống / đã uống nước.
O(T)
BT 7
Talent Show – Tỉ lệ tài năng/trọng lượng
Binary search trên kết quả, mỗi lần kiểm tra bằng 0/1 Knapsack với adjusted talent.
O(N·W·log)
BT 8
Mooriokart – Đường đua
DFS tách cây + Knapsack cộng dồn chiều dài đường đi trong rừng, nhân hoán vị chu trình.
O(N·Y^1.5)
